# 환불 정책

쿠폰 환불과 에스크로(온라인 의뢰) 환불을 포괄하는 통합 환불 정책. 프론트엔드에서 환불 관련 UI/UX 구현 시 참고.

## 1. 핵심 원칙

- **쿠폰 환불**: 한 번도 사용하지 않은(AVAILABLE) 쿠폰만 환불 가능. 한 번이라도 티켓에 예약된 이력이 있으면 환불 불가, 재사용만 가능.
- **에스크로 환불**: 작업 진행 상태에 따라 전액/부분/불가로 구분. 완료 확인(또는 자동 완료) 이후에는 환불 불가.
- **오프라인 의뢰**: 플랫폼이 결제에 개입하지 않으므로 당사자 간 직접 협의.

## 2. 쿠폰 환불 정책

### 쿠폰 상태별 환불 가능 여부

| 쿠폰 상태 | 환불 | 재사용 | 설명 |
|---|---|---|---|
| **AVAILABLE** | 가능 | 가능 | 구매/지급 후 한 번도 티켓에 사용하지 않은 쿠폰 |
| **RETURNED** | 불가 | 가능 | 매칭 실패/취소로 돌려받은 쿠폰 (사용 이력 있음) |
| **RESERVED** | 불가 | 불가 | 현재 티켓에 묶여 있는 상태 |
| **CONSUMED** | 불가 | 불가 | 매칭 성공으로 완전히 사용됨 |
| **EXPIRED** | 불가 | 불가 | 유효기간 초과 |
| **REFUNDED** | — | 불가 | 이미 환불 처리된 쿠폰 |

### 쿠폰 상태 흐름

```
AVAILABLE (사용가능, 환불 가능)
    │
    ├──→ RESERVED (티켓 등록 시 예약, 환불 불가)
    │       │
    │       ├──→ CONSUMED (매칭 성공, 환불/재사용 불가)
    │       │
    │       └──→ RETURNED (매칭 실패/취소, 환불 불가, 재사용 가능)
    │               │
    │               └──→ RESERVED (다시 티켓 등록 가능, 반복)
    │
    └──→ REFUNDED (환불 처리)

AVAILABLE / RETURNED ──→ EXPIRED (유효기간 초과 시 자동)
```

### 패키지 부분 환불

패키지로 구매한 경우 **AVAILABLE 상태인 쿠폰만** 환불 대상.

**예시:** 5장 패키지(₩10,000) 구매 후 — 2장 CONSUMED, 1장 RETURNED, 2장 AVAILABLE
→ AVAILABLE 2장만 환불 가능 = ₩4,000 환불 (장당 ₩2,000)
→ RETURNED 1장은 환불 불가, 새 의뢰에 재사용만 가능

### 환불 불가 사유 (RETURNED)

쿠폰으로 티켓을 등록하면 전문가들이 시간과 노력을 들여 제안서를 작성한다. 매칭이 안 됐더라도 서비스가 이미 작동한 것이므로, 해당 쿠폰은 재사용만 허용하고 현금 환불은 제한한다.

### 쿠폰 환불 금액 산정

| 패키지 | 총 가격 | 장당 단가 | 환불 시 장당 환불액 |
|---|---|---|---|
| TRIAL (1장) | ₩3,000 | ₩3,000 | ₩3,000 |
| BASIC (3장) | ₩7,500 | ₩2,500 | ₩2,500 |
| STANDARD (5장) | ₩10,000 | ₩2,000 | ₩2,000 |
| PREMIUM (10장) | ₩15,000 | ₩1,500 | ₩1,500 |

환불 금액 = AVAILABLE 쿠폰 수 × 해당 패키지 장당 단가

## 3. 에스크로 환불 정책 (온라인 의뢰)

### 상태별 환불 가능 여부

| 에스크로 상태 | 환불 | 처리 방법 |
|---|---|---|
| **ESCROW_HELD + 작업 시작 전** | 전액 환불 | 의뢰인 직접 취소 |
| **WORK_IN_PROGRESS (작업 진행 중)** | 협의 필요 | CS 중재 → 진행률에 따라 부분 환불 가능 |
| **DELIVERED (납품 후 7일 이내)** | 협의 필요 | CS 중재 필요 |
| **CONFIRMED (완료 확인 후)** | 불가 | 이미 전문가에게 정산 진행 중 |
| **SETTLED (정산 완료)** | 불가 | 전문가 계좌 입금 완료 |
| **AUTO_CONFIRMED (자동 완료)** | 불가 | 납품 후 7일 무응답으로 자동 정산 |

### 자동 환불 조건

| 조건 | 기간 | 처리 |
|---|---|---|
| 합의서 작업 마감일 초과 + 납품 없음 | 마감일 +3일 | 의뢰인 알림 → 자동 전액 환불 |
| 납품 후 의뢰인 무응답 | 7일 | 자동 구매확인 → 전문가 정산 (환불 아님) |

### 에스크로 환불 시 PG 수수료

- PG 수수료는 환불해도 돌아오지 않는다
- 카드 약 3.6% 등은 환불 시 매몰비용
- PG사 약관에 따라 수수료가 돌아오는 경우도 있으므로 확인 필요

## 4. 분쟁(Dispute) 처리

### 분쟁 발생 조건

의뢰인은 환불을 원하고, 전문가는 정산을 원하는 경우 분쟁이 발생한다.

### 분쟁 처리 절차

1. 의뢰인 또는 전문가가 "분쟁 신고" 버튼 클릭
2. 에스크로 예치금 **동결** (양쪽 모두 접근 불가)
3. CS가 **채팅 내역 확인** (채팅이 곧 리뷰이므로 근거 명확)
4. 중재 결정: 전액 환불 / 부분 환불 / 전문가 정산
5. 토스 API로 해당 처리 실행

### 분쟁 판단 기준

| 상황 | 판단 |
|---|---|
| 전문가가 합의된 범위를 이행하지 않음 | 의뢰인 전액 환불 |
| 결과물이 합의 범위와 다름 | 부분 환불 또는 수정 요청 |
| 의뢰인이 합의 범위 외 요구 | 전문가 정산 |
| 양쪽 주장이 충돌 | 채팅 내역 + 합의서 기반으로 CS 판단 |

## 5. 오프라인 의뢰 환불

오프라인 의뢰(대면 레슨 등)는 결제가 플랫폼을 거치지 않으므로:

- **쿠폰**: 위 쿠폰 환불 정책과 동일 (AVAILABLE만 환불, RETURNED는 재사용만)
- **레슨비**: 당사자 간 직접 협의. 플랫폼은 중재하지 않음
- **자동 완료**: 진행 시작(IN_PROGRESS) 후 48시간 무응답 시 자동 COMPLETED 처리

## 6. 쿠폰 상태값 요약

| 상태 | 환불 | 재사용 | 설명 |
|---|---|---|---|
| AVAILABLE | 가능 | 가능 | 한 번도 티켓에 예약된 적 없는 상태 |
| RESERVED | 불가 | 불가 | 현재 티켓에 묶여 있는 상태 |
| CONSUMED | 불가 | 불가 | 매칭 성공으로 완전히 사용됨 |
| RETURNED | 불가 | 가능 | 매칭 실패/취소로 반환됨 |
| EXPIRED | 불가 | 불가 | 유효기간 초과 |
| REFUNDED | — | 불가 | 환불 처리 완료 |

## 7. 사용자 안내 문구

### 쿠폰 구매 시

> "쿠폰은 의뢰 등록에 한 번도 사용하지 않은 경우에만 환불이 가능합니다. 의뢰 등록 후 매칭에 실패한 쿠폰은 환불은 불가하나, 다른 의뢰에 재사용할 수 있습니다."

### 티켓 등록 시 (쿠폰 예약 시점)

> "이 쿠폰은 의뢰 등록에 사용됩니다. 등록 후에는 매칭 실패 시 쿠폰이 반환되어 재사용할 수 있지만, 현금 환불은 불가합니다."

### 환불 요청 시 (RETURNED 쿠폰)

> "이 쿠폰은 이전에 의뢰에 사용된 이력이 있어 환불이 불가합니다. 다른 의뢰에 다시 사용하실 수 있습니다."

## 8. 환불 관련 UI 화면 안내

| 화면 | 표시 내용 |
|------|----------|
| 쿠폰 구매 페이지 | 구매 시 안내 문구 표시. 환불 정책 링크 |
| 쿠폰 상세/목록 | 각 쿠폰의 상태(AVAILABLE/RETURNED/CONSUMED 등) 뱃지 표시. RETURNED 쿠폰에 "재사용 가능" 안내 |
| 환불 요청 화면 | AVAILABLE 쿠폰만 환불 가능. RETURNED 쿠폰 선택 시 환불 불가 사유 안내 |
| 티켓 등록 쿠폰 선택 | AVAILABLE + RETURNED 쿠폰 모두 사용 가능. 쿠폰 예약 시 안내 문구 |
| 온라인 의뢰 상세 | 에스크로 상태에 따른 환불 가능 여부 안내 |
| 분쟁 신고 | 분쟁 신고 버튼 + 처리 절차 안내 |
| 오프라인 의뢰 상세 | "레슨비는 당사자 간 직접 협의" 안내 |
